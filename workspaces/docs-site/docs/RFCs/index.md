# RFCs (Design records)

RFCs are design/evolution records (PEP-style). They are primarily for contributors and advanced readers.

  <a class="inc-route-card" href="../contributing/how-to/writing_rfcs/"><span class="inc-eyebrow">Propose</span><strong>Write an RFC</strong><span>Use the repository process and template for a design that changes the language, tooling, or architecture.</span></a>
  <a class="inc-route-card" href="../roadmap/"><span class="inc-eyebrow">Plan</span><strong>Read the roadmap</strong><span>See release direction and delivery context. RFC status alone does not promise a release.</span></a>
</div>

!!! info "Design records, not canonical user docs"
    RFCs explain why a design exists and how it evolved. User-facing behavior remains canonical in the Language and Tooling documentation.
