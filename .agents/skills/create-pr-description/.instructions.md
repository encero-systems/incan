# PR Description Generator Skill

This skill generates PR descriptions following the repository's pull request template.

## Usage

When the user asks to create a PR description:

1. Identify the repository (incan/ or IncQL/)
2. Locate the PR template at `.github/pull_request_template.md`
3. Run `git diff main...HEAD` to see all changes
4. Check for associated issue number from branch name (e.g., `bugfix/184-...` → issue #184)
5. Fill in the template sections based on the analysis
6. If an issue number was found, append `Closes #<issue number>` at the end
7. Output the complete PR description

## Key principles

- **Repository-aware**: The skill automatically locates the PR template in the target repository
- **Template-following**: Always use the repository's PR template structure
- **Change-focused**: Base the description on actual git diff changes
- **Issue-aware**: Extract issue number from branch name and append `Closes #<issue number>` when applicable
- **Complete**: Include all template sections with appropriate checkboxes

## Example prompts

- "Create a PR description for bugfix/184 in the incan repo"
- "Generate a PR description for feature/180 in the incan repo"
- "Create a PR description for feature/new-dataset-api in the IncQL repo"
